Vue d'ensemble — Projet 2
🏥 Clinique Pasteur ✅ Infrastructure déployée BTS SIO SISR 2026

Contexte

La Clinique Pasteur souhaite moderniser son infrastructure réseau afin de sécuriser les échanges de données médicales et permettre un accès distant sécurisé aux médecins. Le projet couvre la mise en place d'un serveur web en DMZ, la sécurisation des flux par pare-feu Stormshield, le durcissement des serveurs et la mise en place d'un tunnel VPN SSL.

Périmètre : Serveur web IIS en DMZ, base de données MySQL isolée, pare-feu Stormshield, accès VPN SSL pour les médecins distants.

# Enregistrer PHP comme application FastCGI

  -Value @{fullPath='C:\PHP\php-cgi.exe'}

# Handler mapping pour les fichiers .php

  -PSPath 'IIS:\Sites\Default Web Site' -Name . -Value $handler

Sécurisation HTTPS — Certificat auto-signé
VM 112 — SRV-WEB-DMZ ✅ Terminé

Un certificat auto-signé est généré via PowerShell et lié au site IIS sur le port 443. Il garantit le chiffrement des échanges entre les postes du LAN et le serveur web.

Génération du certificat

PowerShell# Générer un certificat auto-signé valide 2 ans
$cert = New-SelfSignedCertificate `
  -DnsName "clinique-pasteur.local", "10.0.11.22" `
  -CertStoreLocation "cert:\LocalMachine\My" `
  -NotAfter (Get-Date).AddYears(2) `
  -KeyUsage DigitalSignature, KeyEncipherment `
  -FriendlyName "Clinique Pasteur HTTPS"

Write-Host "Thumbprint : $($cert.Thumbprint)"

Liaison HTTPS sur IIS

PowerShellImport-Module WebAdministration

# Créer le binding HTTPS sur le port 443
New-WebBinding -Name "Default Web Site" `
  -Protocol "https" -Port 443 -IPAddress "*"

# Associer le certificat
$binding = Get-WebBinding -Name "Default Web Site" -Protocol "https"
$binding.AddSslCertificate($cert.Thumbprint, "My")

# Vérification
netsh http show sslcert ipport=0.0.0.0:443
Note : Les navigateurs affichent un avertissement pour les certificats auto-signés. En production, utiliser une CA interne (AD CS) ou un certificat public.

Nginx — Reverse proxy & Load balancing
server-nginx — 10.0.11.30 🔲 Planifiée

Nginx est configuré en round-robin : chaque requête est transmise alternativement à nodejs-1 puis nodejs-2. En cas de panne, le trafic est automatiquement redirigé.

Installation

Bashsudo apt update && sudo apt install nginx -y
sudo systemctl enable --now nginx

Configuration reverse proxy

Nginx# /etc/nginx/sites-available/clinique-api

upstream nodejs_backend {
    server 10.0.11.31:3000;
    server 10.0.11.32:3001;
    # Round-robin par défaut — pas de directive nécessaire
}

server {
    listen 80;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name clinique-api.local;

    ssl_certificate     /etc/nginx/ssl/clinique.crt;
    ssl_certificate_key /etc/nginx/ssl/clinique.key;
    ssl_protocols       TLSv1.2 TLSv1.3;

    location / {
        proxy_pass         http://nodejs_backend;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_connect_timeout 5s;
    }
}
Bashsudo ln -s /etc/nginx/sites-available/clinique-api /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

HTTPS sur Nginx — Certificat OpenSSL auto-signé
server-nginx — 10.0.11.30 🔲 Planifiée

Génération d'un certificat SSL auto-signé via OpenSSL pour sécuriser les communications entre les clients et le reverse proxy Nginx.

Génération du certificat

Bashsudo mkdir -p /etc/nginx/ssl

sudo openssl req -x509 -nodes -days 730 \
  -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/clinique.key \
  -out    /etc/nginx/ssl/clinique.crt \
  -subj "/C=FR/ST=HautsPyrenees/L=Tarbes/O=Clinique Pasteur/CN=clinique-api.local" \
  -addext "subjectAltName=IP:10.0.11.30,DNS:clinique-api.local"

sudo chmod 600 /etc/nginx/ssl/clinique.key

# Vérifier le certificat
openssl x509 -in /etc/nginx/ssl/clinique.crt -noout -text | grep -E "Subject:|IP:"

Script de test automatisé

Bash#!/bin/bash
PASS=0; FAIL=0
for i in $(seq 1 20); do
  code=$(curl -sk -o /dev/null -w "%{http_code}" https://10.0.11.30/health)
  [ "$code" = "200" ] && PASS=$((PASS+1)) || FAIL=$((FAIL+1))
done
echo "Résultat : $PASS/20 réussies"
[ $FAIL -eq 0 ] && echo "✅ TOLÉRANCE OK" || echo "❌ BASCULEMENT INCOMPLET"
Critère : 20/20 requêtes réussies lors du test de basculement = tolérance aux pannes validée.
